# 项目结构说明

```

│   └── index.html         # Web界面
├── scripts/                # 脚本和工具
│   ├── python/            # Python数据库脚本
│   │   ├── check_db.py    # 检查数据库内容
│   │   └── reset_db.py    # 重置数据库
│   ├── data-processing/   # 数据处理脚本
│   │   ├── convert_to_api_format.py
│   │   ├── parse_answer2_to_json.py
│   │   └── parse_answers_to_json.py
│   └── tests/             # 测试脚本
│       ├── test_api_integration.py
│       ├── test_database_operations.py
│       ├── test_file_parsing.py
│       ├── test_performance.py
│       └── run_all_tests.py
├── docs/                   # 文档和参考资料
│   ├── API接口文档.md      # API接口文档
│   └── reference/         # 参考资料和数据
│       ├── answer2.txt
│       ├── answer2_api_format.json
│       ├── answer2_parsed.json
│       ├── answers.txt

│   └── workflows/
├── target/                 # Rust编译输出
├── questions.db           # SQLite数据库文件
```

## 脚本说明

### 数据库脚本 (scripts/python/)
- `check_db.py`: 检查数据库中的内容和结构
- `reset_db.py`: 重置数据库，删除所有数据

### 数据处理脚本 (scripts/data-processing/)
- `convert_to_api_format.py`: 将数据转换为API格式
- `parse_answer2_to_json.py`: 解析答案文件2为JSON格式
- `parse_answers_to_json.py`: 解析答案文件为JSON格式

### 测试脚本 (scripts/tests/)
- `test_api_integration.py`: API集成测试
- `test_database_operations.py`: 数据库操作测试
- `test_file_parsing.py`: 文件解析测试
- `test_performance.py`: 性能测试
- `run_all_tests.py`: 运行所有测试

## 使用说明

1. **运行项目**:
   ```bash
   cargo run
   ```

2. **检查数据库**:
   ```bash
   python scripts/python/check_db.py
   ```

3. **重置数据库**:
   ```bash
   python scripts/python/reset_db.py
   ```

4. **运行测试**:
   ```bash
   python scripts/tests/run_all_tests.py
   ```
